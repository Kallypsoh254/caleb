Technical Opinions

Why I Chose X over Y

Opinionated decisions backed by production experience — not hype.

Go
vs
Node.js

For APIs handling 10k+ RPS, Go's goroutines eliminate GC pauses, giving predictable sub-5ms p99 latency that Node.js event loop can't match under sustained load.

PostgreSQL
vs
MongoDB

For structured business data, Postgres JSONB + strong consistency + mature tooling beats MongoDB's flexibility. Schema discipline prevents the data swamps I've seen at scale.

Terraform
vs
CloudFormation

For multi-cloud IaC, Terraform's provider ecosystem and state management beat CloudFormation's AWS lock-in. HCL is also significantly more readable than YAML stacks.

Kafka
vs
RabbitMQ

For event streaming at scale, Kafka's log-based retention and consumer group model enable replay, time-travel debugging, and fan-out that RabbitMQ's queue model can't do.

Next.js
vs
CRA

For production React apps, Next.js server components, streaming SSR, and built-in image optimization deliver 40%+ better Core Web Vitals than Create React App out of the box.

Rust
vs
C++

For systems programming, Rust's ownership model eliminates entire classes of memory bugs at compile time, giving C++ performance with dramatically better safety guarantees.
